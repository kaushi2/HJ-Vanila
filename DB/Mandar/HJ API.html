    HJ API v0.6

    Authentication

    To authorize, use this code:

    Make sure to replace auth_code with user Token key.

    Authorization: JWT auth_code

    You must replace auth_code with user Token key returned by login method.

    Login

    POST /api/auth/login/

    Check the credentials and return the REST Token if the credentials are valid and authenticated. Calls login method to register User ID in session framework Accept the following POST parameters: username, password Return the REST Framework Token Object’s key.

     curl -X POST -d "username=admin&password=password123" <login_url>

 > Returns

 {
    "token": "auth_code",
    "user": {
        "pk": user_id,
        "username": "username",
        "email": "email@abc.com",
        "first_name": "John",
        "last_name": "Doe"
    }
}

    Request Parameters

    Field Required
    username true
    password true
    email false

    Logout

    POST /api/auth/logout

    Delete the Token object assigned to the current User object.

     curl -X POST -d "" <logout_url>

 > Returns

{
    "detail":"Successfully logged out."
}

> If an API is called made after logout then an error would be returned

{
  "detail":"Signature has expired."
}

    Register

    POST /api/registration/

    User Registration

     curl -X POST -d "username=admin@gmail.com&password1=password123&password2=password123&email=admin@gmail.com" <registration_url>

 > Returns

 {
    "token": "auth_code",
    "user": {
        "pk": user_id,
        "username": "username",
        "email": "email@abc.com",
        "first_name": "John",
        "last_name": "Doe"
    }
}

    Request Parameters

    Field Required
    username true
    email true
    Password1 true
    Password1 true

    Verify Email

    POST /api/registration/verify-email/

    Forgot Password

    POST /api/auth/password/reset/

    Password Reset Confirm

    POST /api/auth/password/reset/confirm/

    Change Password

    Check the credentials and return the REST Token if the credentials are valid and authenticated. Calls login method to register User ID in session framework Accept the following POST parameters: username, password Return the REST Framework Token Object’s key.

    401 Unauthorized – Your API key is wrong

    404 Not Found – The specified kitten could not be found
    405 Method Not Allowed – You tried to access a kitten with an invalid method

    410 Gone – The kitten requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many kittens! Slow down!
